## What is Lcurve?

Lcurve is a general-purpose MATLAB function for evaluating and using the **L-curve criterion** in Tikhonov regularization problems. Given an ill-posed linear system `A*x ≈ b`, you need to choose a regularization parameter `λ` that balances data fidelity against solution smoothness. The L-curve criterion provides a principled, data-driven way to make that choice.

Key capabilities:

- **Analytic curvature**: curvature of the log-L-curve is computed in closed form — no finite differences, no approximation errors.
- **Automatic knee detection**: finds the optimal `λ` automatically by locating the maximum-curvature point of the log-L-curve.
- **Multiple solver backends**: SVD, backslash (`\`), symmetrized backslash, `inv`, `decomposition`, and more.
- **Arbitrary regularization operators**: supports any matrix `L`, not just the identity.
- **Warm-up mode**: pre-computes the SVD of `A` once and reuses it across many evaluations.

## Core workflow

```matlab MATLAB
% 1. Warm up A (pre-computes SVD once)
A = Lcurve(A);

% 2. Evaluate the L-curve over a range of lambdas
LAMBDAs = 10.^(-20:1:10);
XYKCs = Lcurve(A, b, L, LAMBDAs);

% 3. Find the optimal lambda automatically
LAMBDA_knee = Lcurve(A, b, L);

% 4. Solve at the knee
[x_best, K_knee] = Lcurve(A, b, L);
```

## Why analytic curvature?

Most L-curve implementations estimate curvature using finite differences over a discrete set of `λ` values, which introduces numerical noise and requires a dense grid. Lcurve computes curvature **analytically** using matrix calculus — giving exact values at any single `λ` with no grid required.

<Note>
The analytic approach means you can accurately locate the knee even with a coarse initial `λ` grid, then zoom in automatically. See the [knee detection guide](/guides/knee-detection) for details.
</Note>
